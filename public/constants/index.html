Odin by Example: Constants

Constants are values that cannot be changed after declaration. Unlike variables, they are evaluated at compile time.

In Odin, constants are declared with the :: operator and can hold strings, booleans, or numeric values.

The compiler can infer the type from the value, but you can also specify types explicitly when needed.

$ odin run constants.odin -file
constant
6e+11
600000000000
-0.28482117633414604

Constant declarations can be grouped in a block for better organization. Use #enum for auto-incrementing integer constants and #bit_field for bitmask values.

Monday = 0 Sunday = 6 Read = 1 Write = 2 Execute = 4 .Read|.Write = {Read, Write}

package main

import "core:fmt"

// Single constants can be declared at package scope.
s :: "constant"
n :: 6e11
d :: 3e20 / n

// Group related constants with #enum for sequential values.
Days :: enum {
	Monday,
	Tuesday,
	Wednesday,
	Thursday,
	Friday,
	Saturday,
	Sunday,
}

// Use #bit_field for bitmask-style constants.
Permissions :: bit_field[uint] {
	Read    = 1,
	Write   = 2,
	Execute = 4,
}

main :: proc() {
	fmt.println(s)

	// Constants can be used in arithmetic expressions.
	fmt.println(d)

	fmt.println(int(d))

	// Numeric functions work with constants.
	fmt.println(-1.56)

	// #enum constants can be cast to int.
	fmt.println("Monday   =", int(Monday))
	fmt.println("Sunday   =", int(Sunday))

	// #bit_field constants hold their assigned values.
	fmt.println("Read     =", Permissions.Read)
	fmt.println("Write    =", Permissions.Write)
	fmt.println("Execute  =", Permissions.Execute)

	// Bitmask values can be combined with set notation.
	fmt.println(".Read|.Write =", Permissions{.Read, .Write})
}
